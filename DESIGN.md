# Design System: 造梦空间

**Project ID:** local-react-mindspace

## 1. Visual Theme & Atmosphere

造梦空间是一段可操控的抽象短片，不是星空主题的问卷，也不是冥想产品。空间的核心材料是一层会被拉扯、压陷、起皱并慢慢复原的暗色薄膜。一条暖杏色细缝始终藏在膜下，代表用户仍然在寻找希望的部分。

设计参数为：

- **Design variance 9:** 构图偏离中心，文字退到边缘，互动材料占据主要画面。
- **Motion intensity 9:** 动效来自输入和状态变化，不使用没有含义的持续旋转。
- **Visual density 2:** 每幕只保留一句引导、一个主要动作和必要的安全出口。

整体气质是煤灰、矿物、安静、有重量、可复原。强烈输入带来更宽更慢的形变，不出现闪烁、爆炸或突然的高对比刺激。

## 2. Color Palette & Roles

- **Coal Field, `#070809`, `oklch(0.14 0.008 240)`:** 页面唯一背景，建立无边界黑场。
- **Mineral Surface, `#171B19`, `oklch(0.22 0.012 165)`:** 薄膜与近景层，不作为卡片背景。
- **Fog Layer, `#4E5752`, `oklch(0.43 0.018 165)`:** 低对比轮廓、远景和非文本材料。
- **Living Trace, `#F08A72`, `oklch(0.72 0.14 35)`:** 唯一品牌强调色，只用于希望细缝、焦点、选中和主要状态变化。
- **Soft Ink, `#F2EFE9`, `oklch(0.95 0.01 80)`:** 主要文字。
- **Quiet Ink, `#B9B8B2`, `oklch(0.76 0.01 90)`:** 次要说明，必须保持可读对比度。
- **Safety Signal, `#F0A096`, `oklch(0.78 0.10 25)`:** 仅用于真人帮助和危机信息，属于语义状态，不参与装饰。

禁止同时使用蓝色、金色、玫红为情绪编码。情绪差异由黏度、张力、重量、回弹和边界清晰度表达。

情绪物理规则：

- **悲伤:** 高黏度、缓慢下坠、边界向外扩散。
- **生气:** 高张力、短促剪切、快速但受控的回弹。
- **恨意:** 边界锐利、惯性偏强，但停止后不继续向外扩张。
- **不甘:** 持续牵引、拉长后多次归位，回弹时间较长。
- **无奈:** 形态松散、响应延迟、低幅度复原。
- **开心:** 表面变轻、空间展开、暖色细缝更容易显露。
- **说不清:** 轮廓互相渗透，不强迫聚合成单一形状。

## 3. Typography Rules

- **界面与标题:** `"PingFang SC", "Noto Sans SC", system-ui, sans-serif`。标题使用 300-400 字重，不使用巨大宋体制造情绪。
- **结尾保留句:** 可使用 `"Songti SC", "Noto Serif SC", serif`，且全程只出现一次，表示从功能空间回到人的语言。
- **标题尺寸:** 桌面 40-64px，手机 34-48px。字距不小于 `-0.04em`。
- **功能文字:** 12-14px，短句，正常字距。按钮和标签不使用展示字体。
- **文案原则:** 先给动作，再给解释。每幕可见引导不超过 20 个汉字。

## 4. Component Stylings

* **主要动作:** 不使用矩形 CTA。主要动作附着在互动材料本身，例如按住细缝、拖动薄膜、选择应力纹理上的词。
* **功能按钮:** 透明背景、清晰文字、最小 44px 命中区；悬停和焦点使用 Living Trace 细线。按钮按下缩放到 0.98。
* **安全入口:** 固定在右上角，视觉安静但始终可见，不依赖颜色作为唯一提示。
* **输入:** 标签始终位于输入上方。输入只用一条底线，不使用浮层卡片。占位文字提供示例而不是承担标签职责。
* **容器:** 不使用玻璃卡片和圆角面板。层级由景深、留白和薄膜明暗表达。
* **圆形:** 不作为默认组件形状。只有真实的点状锚点可以接近圆形，核心意象必须是不规则且可形变的。

## 5. Layout Principles

- 主画面使用全视口黑场，界面文字与互动材料形成偏心构图。
- 标题通常位于左上或左侧三分之一，不与互动材料争夺中心。
- 顶部不显示装饰性品牌点、英文副标或可见进度轨道。进度保留无障碍语义，由暖色细缝在画面中的长度隐性表达。
- 桌面互动材料可跨越 65-80vw；手机端回到单列，但材料仍保持偏心，不缩成居中小球。
- 安全入口、减弱动态和本地隐私说明使用固定功能层，不参与场景动画。

## 6. Stage Material Map

| Stage | Material state | User action | Meaning |
| --- | --- | --- | --- |
| Arrival | 一道很窄的暖色细缝 | 按住 | 空间开始回应，而不是要求用户说明 |
| Calibration | 被拉扯的暗色薄膜 | 自由拖动，确认后停下 | 身体方向先于语言 |
| Confirmation | 应力纹理上浮出的词 | 靠近一个词 | Agent 只提供候选，命名权属于用户 |
| Expression | 可被划开又会复原的表面 | 划动、按住、出声 | 情绪可以存在和移动，不需要马上消失 |
| Embodiment | 退入景深的结 | 调整距离 | 心事仍在，但不必贴住自己 |
| Values | 膜下显露的暖色脉络 | 将一条脉络接入情绪痕迹 | 用户仍然在乎的事成为方向 |
| Closure | 细缝延展成地平线 | 留下一个小行动 | 希望不是外部救命稻草，而是自身仍在行动 |
| Help | 静止、清晰、无沉浸干扰 | 直接联系真人 | 安全优先于体验完整性 |

## 7. Motion Rules

- 同一个 Canvas 在阶段间持续存在，保留用户留下的 `trace`、`tension` 和 `settledness`。
- 普通过渡 180-260ms；材料复原 1.2-4s；低频呼吸 6-8s。
- 只动画 transform、opacity 和 Canvas 绘制参数，不用 CSS 动画 width 或 height。
- 强输入扩大形变范围并延长复原时间，不提高闪烁频率。
- 麦克风电平需平滑、限幅，不能造成突然爆闪。
- `prefers-reduced-motion` 或用户手动减弱动态时，材料保持静态形态，只用短交叉淡化反馈状态。
- 校准阶段允许探索，第一次松手不会自动跳转。达到最小位移后再显现确认动作。
- 表达阶段只有按住或捕获指针时才记录释放轨迹，普通鼠标经过不算表达。

## 8. Accessibility And Safety

- 所有主要动作必须能用键盘完成，焦点边界使用 Living Trace。
- 可见文字达到 WCAG AA；功能按钮命中区至少 44px。
- Canvas 仅为视觉反馈，关键状态通过语义 HTML 和 ARIA 表达。
- 危机表达立即退出沉浸场景，显示 12356、120、110 和清晰说明。
- 麦克风只在本地读取音量，不录音、不转写、不上传。
- 明日行动不是完成流程的强制门槛。先提供低负担行动意象，再允许用户自定义。
- 禁止闪烁、快速亮度跳变、无法停止的高频运动和用颜色单独表达情绪。

## 9. Patterns To Retire

- 透视星点、宇宙穿梭和深蓝加暖金的疗愈类默认视觉。
- 每一幕都出现的球体、圆环、外发光和居中大标题。
- 可见的渐变进度条、装饰性状态点和英文副标。
- 仅通过换颜色区分悲伤、生气、不甘等情绪。
- 切换场景时抹掉用户上一幕留下的全部痕迹。
